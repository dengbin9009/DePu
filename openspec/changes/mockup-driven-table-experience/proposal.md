# 变更提案：效果图驱动的牌桌体验优化

## 背景

当前多人德州扑克已经具备账号、钱包、房主建房、邀请码加入、socket 实时同步、行动倒计时、动作日志、聊天、房间战绩榜和正式手牌回放等基础能力。但前端体验仍偏“工程验证页”：建房配置较粗、牌桌视觉密度不足、入座买入需要跳转到玩家页，聊天/战绩/牌谱/设置等能力缺少移动端游戏桌常见的抽屉式承载。

本次需求以用户提供的九张效果图为参考，目标是把现有多人牌桌优化成更完整的移动端游戏桌体验：玩家可以按档位创建训练赛房间，在牌桌内坐下并选择带入记分牌，随时打开商城补充金币，使用底部菜单进入聊天、当前战绩、牌谱回顾和牌桌设置，并在等待开局或房主开局时看到与效果图一致的核心信息结构。

## 目标

- 将创建房间体验升级为“创建比赛”配置页，支持玩法、短牌模式、房间名称、Ante、买入范围、带入上限、训练时长和单桌人数等可理解配置。
- 将金币充值包装为“商城”页面，优先落地金币分类和现有模拟充值能力，保留钻石、VIP 卡、装扮、道具、课程等分类占位。
- 在牌桌内提供“补充记分牌”买入弹窗，用户坐下前可按房间规则选择带入记分牌，金币不足时引导前往商城。
- 将正式牌桌重构为移动端游戏桌布局，包含座位环、房间信息、邀请码、网络延时、等待/开局按钮、底部工具栏和玩家自己的座位信息。
- 将聊天、当前战绩、牌谱回顾和牌桌设置做成牌桌内覆盖面板或底部抽屉，减少路由跳转造成的牌局割裂。
- 保留现有 socket、正式手牌结果、战绩榜、聊天和回放数据作为权威来源，前端不得自行推断影响牌局结果的规则。

## 非目标

- 不接入真实支付、退款、支付回调或运营后台。
- 不实现好友系统、邀请好友真实社交关系、私聊或通讯录。
- 不实现语音聊天、麦克风采集、实时语音房或语音审核。
- 不实现装扮、道具、课程、VIP 权益、钻石、积分的完整交易和权益系统。
- 不实现 SNG 完整赛事编排、盲注升级淘汰赛、排行榜赛季或跨房间比赛系统。
- 不实现保位离座、降落伞、投诉、收藏的完整后端流程；本次可做说明、占位或受限入口。
- 不移除独立规则测试页，不把规则测试页调试能力并入正式多人牌桌。

## 用户价值

- 房主创建房间时能直观看到关键牌桌参数，而不是填写裸数字。
- 玩家在牌桌内即可完成坐下、买入、金币不足补充和返回牌桌，流程更像真实产品。
- 房间等待开局时，玩家能看到邀请码、玩法、房号、Ante、级别和当前手数，便于邀请和确认牌桌配置。
- 聊天、战绩、牌谱、设置不再挤在侧栏中，移动端操作更自然。
- 商城把现有模拟充值能力变成清晰入口，后续若接真实支付或其他资产类型也有可扩展位置。

## 风险

- 效果图包含不少超出现有 v1/v1.1 的功能入口，需要明确哪些真实落地、哪些仅占位，避免范围膨胀。
- 创建比赛配置中的 Ante、买入范围、训练时长和人数需要与后端房间规则对齐，否则会出现 UI 可选但服务端不支持的配置。
- 牌谱回顾若做成牌桌内面板，必须继续遵守正式回放的公开信息边界，不能泄露未公开底牌。
- 商城虽然仍是模拟充值，但金额、赠送金币和资产展示容易让用户误解为真实支付，需要明确“模拟充值”语义。
- 移动端牌桌布局元素多，必须避免文字重叠、座位遮挡、底部工具栏挡住操作按钮。

## 成功标准

- 用户能从创建比赛页完成短牌训练赛房间创建，并进入牌桌看到与配置一致的房间信息。
- 用户点击空座或坐下入口后，在牌桌内完成带入记分牌选择；金币不足时能跳转商城充值后返回继续坐下。
- 房主在等待状态下能看到“解散比赛、邀请好友、开始”操作；非房主不展示或禁用房主专属操作。
- 牌桌底部工具栏能打开聊天记录、当前战绩、牌谱回顾和牌桌设置，关闭后回到同一牌桌状态。
- 商城金币页能展示当前金币余额和充值档位，并复用现有模拟充值能力更新钱包。
- 现有多人 socket 同步、开局、行动、结算、战绩榜、聊天、正式回放和独立规则测试页继续可用。
