# 快速开始：德州扑克完整牌局模拟器

> 当前文档描述第一版目标使用方式。代码实现完成后，命令需要与实际项目脚本保持一致。

## 前置条件

- Go 1.22 或更高版本
- Node.js 20 或更高版本
- SQLite 可由后端驱动自动创建本地数据库文件

## 启动后端

```bash
cd backend
go test ./...
go run ./cmd/depu-server
```

默认 API 地址：

```text
http://localhost:8080
```

## 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认前端地址：

```text
http://localhost:5173
```

## 验证核心流程

1. 打开前端页面。
2. 创建一桌 4 人长牌牌局，选择 `blinds` 下注结构，设置按钮位、小盲和大盲。
3. 确认玩家座位号为 `1..N` 连续整数，按钮位对应已有座位，玩家名称非空且本局唯一。
4. 使用随机发牌开始牌局。
5. 按页面显示的合法动作推进到翻牌、转牌、河牌和摊牌。
6. 查看结算结果和行动历史。
7. 请求回放到某个动作序号，确认返回的是只读快照，且不改变当前牌局权威状态。
8. 请求 `toSeq=0`，确认返回初始发牌后的只读快照。
9. 请求大于最新动作序号的 `toSeq`，确认系统返回错误且不改变当前权威状态。
10. 在历史面板手动切换两个动作节点，确认牌桌显示对应只读快照，可有短暂过渡动画，但不存在整手牌自动播放时间轴入口。
11. 刷新页面后重新打开同一牌局，确认状态和历史仍可读取。

## 验证短牌 ante 流程

1. 创建一桌短牌牌局，选择 `ante` 下注结构。
2. 设置正整数 `ante` 和 `buttonBlind`；确认二者不要求固定倍数或大小关系。
3. 确认所有玩家翻牌前本轮已投入额包含实际支付的 `ante`。
4. 确认按钮位额外包含实际支付的 `buttonBlind`，当前最高投入为 `ante + buttonBlind` 或短筹码实际最高投入。
5. 确认非按钮且未全下玩家需要补齐到 `ante + buttonBlind`。
6. 确认翻牌前由按钮左侧最近的可行动玩家先行动。

## 验证短筹码强制投入

1. 创建一桌短牌 `ante` 牌局，使至少一名玩家筹码不足以完整支付 `ante` 或按钮位不足以完整支付 `ante + buttonBlind`。
2. 确认该玩家以剩余筹码支付强制投入，并立即进入全下状态。
3. 推进到摊牌，确认主池和边池按实际投入金额拆分。
4. 对长牌 `blinds` 重复类似流程，确认小盲或大盲短筹码也按实际剩余筹码全下。

## 验证创建校验

1. 尝试创建 `smallBlind >= bigBlind` 的 `blinds` 牌局，确认系统拒绝。
2. 尝试创建 `ante=0` 或 `buttonBlind=0` 的短牌 `ante` 牌局，确认系统拒绝。
3. 尝试使用跳号座位、重复座位、按钮位不存在、空玩家名或重复玩家名，确认系统拒绝。
4. 确认系统不会自动改写非法输入后继续创建。

## 验证调试模式

1. 创建一桌短牌调试牌局，选择 `ante` 下注结构。
2. 在首个玩家动作前只指定部分玩家手牌和部分公共牌。
3. 确认系统拒绝重复牌和短牌非法点数，例如 `2s`。
4. 推进牌局，确认已指定牌保持固定，未指定手牌和后续公共牌由后端从剩余合法牌堆随机补足。
5. 使用 A-6-7-8-9 相关牌面，确认系统识别短牌顺子。
6. 提交任一玩家动作后再次尝试修改调试牌，确认系统拒绝，且 v1 不提供重置到调试编辑状态的入口。

## 验证牌桌视觉和扑克动画

1. 使用 1280px 宽桌面视口打开前端，创建 4 人牌局。
2. 确认第一屏以手机优先牌桌为主视觉，能同时看到玩家座位、所有玩家手牌、公共牌区域、筹码/底池、按钮位或当前行动者和行动按钮。
3. 确认主视角玩家固定在底部专区，玩家名称、筹码/投入、行动按钮和两张大手牌不被任何座位或卡牌遮挡；其余 3 名玩家分布在上方和左右侧，并在座位内显示两张小手牌。
4. 确认翻牌前玩家座位不显示当前牌型标签。
5. 推进到翻牌圈，确认新公共牌以发牌或翻牌动画进入牌桌中心，动画在 1 秒内完成，且不遮挡行动控制；同时确认未弃牌玩家座位显示由当前公共牌和手牌组成的当前最佳牌型。
6. 连续提交两个合法动作，确认当前行动者高亮随行动权移动，上一行动者恢复普通状态。
7. 当合法动作包含下注或加注时，确认底部行动区显示金额滑轨、数字输入和“最小 / 半池 / 底池 / 全下”快捷按钮；拖动滑轨或输入数字后，下注/加注按钮文字同步显示选中金额，提交后历史记录中的动作金额与选中金额一致。
8. 分别创建 2 人、6 人、9 人和 10 人牌局，确认座位布局都可读；4 人桌在 1280px 桌面第一屏完整可见，6 到 10 人桌可以更紧凑。
9. 切换到移动窄屏视口，确认玩家座位、卡牌、底池、下注滑轨和控制区不发生不可理解的重叠；6 到 10 人桌允许纵向滚动，关键文字仍可读。
10. 启用浏览器减少动态效果偏好或模拟动画不可用，确认牌桌降级为静态状态后仍完整展示当前局面。
