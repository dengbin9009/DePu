## 依赖顺序

- Phase 1 必须先完成。
- Phase 2 阻塞所有用户故事。

- US3 和 US4 可在基础状态机完成后并行推进。
- US5 可在规则集模型稳定后推进。

## MVP 策略

1. 完成项目初始化和基础领域模型。
2. 完成 US1 创建和推进牌局。
3. 完成 US2 规则判定和结算。
4. 停下验证 MVP：随机长牌牌局能完整结束，核心规则测试通过。
5. 再加入调试模式、保存回放和规则集展示。
